# Wallet API Service

## Структура

### model (сквозные данные)

Сущности проекта:
- Кошелёк
- Операция двжения средств

Структуры, описывающие сущности, которыми пользуются все слои для передачи данных между собой.

### handler (транспортный слой)

Маршруты и ручки. Действия, которые пользователь может совершать:
- Получение баланса кошелька
- Внесение средств на кошелёк
- Списание средств с кошелька
- Перевод средств между кошельками

### repository (слой данных)

Взаимодействие сервиса с хранилищем данных:
- БД PostgreSQL

## Жизненный цикл запроса

1. Запрос поступает по протоколу HTTP
2. `handler` отправляет запрос в соответствующий `service`
3. `service` валидирует входные данные
4. Если входные данные невалидны, то `service` возвращает в `handler` ошибку
5. Если входные данные   валидны, то `service` формирует объект данных `model`
6. `service` запрашивает выполнение необходимых операций с данными `domain` в базе данных через соответсттвующий `repository`
7. `service` получает ответ от `repository` и отадёт его в `handler`
8. `handler` формирует ответ и отдаёт его пользователю
